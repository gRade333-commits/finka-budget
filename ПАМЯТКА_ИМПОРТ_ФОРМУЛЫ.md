# 📋 Памятка: Импорт данных и управление формулами

## 🎯 Главная идея

После массового импорта данных из Excel:
- ✅ **Все данные сохраняются** во всех филиалах
- 🔒 **Автоматические формулы консолидации временно отключены**
- 🔓 **Формулы активируются** при первом ручном редактировании любой ячейки

## 🔄 Как это работает

### 1️⃣ **Массовый импорт**
Когда вы импортируете данные через [import-excel.html](import-excel.html):
- Данные загружаются во все филиалы (Алматы, Уральск, Астана)
- Устанавливается специальный флаг `rb_svodnaya_import_flag`
- Формулы консолидации блокируются автоматически

### 2️⃣ **Работа с данными после импорта**
При открытии [rb-svodnaya.html](rb-svodnaya.html):
- Вы увидите **желтое предупреждение** вверху страницы:
  ```
  🔒 Данные импортированы - автоматические формулы консолидации отключены.
  Начните редактировать любую ячейку для активации формул.
  ```
- Данные отображаются корректно
- При переключении на режим "Консолидировано" - пересчет НЕ происходит

### 3️⃣ **Активация формул**
Когда вы начинаете **редактировать любую ячейку вручную**:
- Флаг импорта автоматически сбрасывается
- Желтое предупреждение исчезает
- Формулы консолидации снова начинают работать
- В консоли появляется сообщение: `🔓 Флаг импорта сброшен - формулы активированы`

## 💡 Зачем это нужно?

### Проблема
После массового импорта формулы пытались пересчитать все данные автоматически, что могло:
- Перезаписать импортированные значения
- Создать несоответствия в данных
- Вызвать путаницу у пользователей

### Решение
Теперь импортированные данные "защищены" до тех пор, пока пользователь:
- Не проверит импортированные данные
- Не начнет ручное редактирование (если нужно)
- Только тогда формулы включатся и будут работать как обычно

## 🛠️ Технические детали

### Флаг в localStorage
- **Ключ**: `rb_svodnaya_import_flag`
- **Значение**: `'true'` после импорта, удаляется при редактировании

### Затронутые функции
1. **import-excel.html**: устанавливает флаг при импорте rb-svodnaya
2. **rb-svodnaya.html**:
   - `loadData()` - проверяет флаг при загрузке
   - `loadConsolidatedData()` - пропускает пересчет если флаг установлен
   - `handleInput()` - сбрасывает флаг при редактировании
   - `updateImportNotice()` - управляет отображением уведомления

## ⚠️ Важно

1. **Флаг сбрасывается при ЛЮБОМ редактировании** - даже если вы просто кликнули в поле и ничего не изменили
2. **Флаг общий для всех филиалов** - сброс происходит один раз для всей системы
3. **После сброса флага формулы работают постоянно** до следующего импорта

## 🎓 Примеры использования

### Сценарий 1: Полный импорт
1. Импортируете Бюджет РБ 2026.xlsx
2. Данные загружаются во все филиалы
3. Открываете rb-svodnaya.html → видите предупреждение
4. Проверяете данные в разных филиалах
5. Начинаете корректировку → предупреждение исчезает
6. Формулы работают нормально

### Сценарий 2: Повторный импорт
1. У вас уже есть данные и работающие формулы
2. Делаете повторный импорт (обновленные данные)
3. Флаг снова устанавливается
4. Формулы блокируются
5. Проверяете новые данные
6. При редактировании флаг сбрасывается

### Сценарий 3: Работа без импорта
1. Вводите данные вручную с самого начала
2. Флаг никогда не устанавливается
3. Формулы работают всегда
4. Всё как раньше

## 🔍 Диагностика

Если формулы не работают:
1. Откройте консоль браузера (F12)
2. Проверьте наличие сообщений о флаге импорта
3. Вручную проверьте флаг:
   ```javascript
   localStorage.getItem('rb_svodnaya_import_flag')
   ```
4. При необходимости сбросьте вручную:
   ```javascript
   localStorage.removeItem('rb_svodnaya_import_flag')
   location.reload()
   ```
