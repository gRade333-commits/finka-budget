## 🚀 Применение изменений ко всем файлам

### Автоматический способ (рекомендуется):

1. **Запустите PowerShell скрипт:**
   ```powershell
   .\add-editable-tables.ps1
   ```

2. **Дождитесь завершения:**
   - Скрипт обновит 15 файлов
   - Покажет прогресс для каждого файла
   - Выведет итоговый отчет

3. **Готово!** Все файлы обновлены

### Ручной способ:

Если хотите обновить файлы вручную:

1. **Откройте HTML файл в редакторе**

2. **Добавьте скрипт в `<head>`:**
   ```html
   <script src="editable-table.js"></script>
   ```

3. **Добавьте инициализацию после загрузки данных:**
   ```javascript
   document.addEventListener('DOMContentLoaded', () => {
       loadData(); // ваша функция загрузки
       
       setTimeout(() => {
           initAllEditableTables('table', handleInput);
       }, 100);
   });
   ```

4. **Сохраните файл**

## ⚠️ Важные моменты

### ✅ Безопасность данных:
- Данные сохраняются в localStorage браузера
- Каждый филиал хранится отдельно
- Автосохранение после каждого изменения

### 🔄 Совместимость:
- Работает со всеми существующими функциями
- Не ломает импорт из Excel
- Не влияет на консолидацию данных
- Сохраняет работу формул

### ⚡ Производительность:
- Мгновенное сохранение (debounce 1 сек)
- Оптимизировано для больших таблиц
- Не замедляет работу страницы

## 🆘 Помощь

### Проблема: Кнопки не появились
**Решение:**
1. Проверьте, подключен ли `editable-table.js`
2. Откройте консоль (F12) и проверьте ошибки
3. Убедитесь, что инициализация вызывается после загрузки данных

### Проблема: Ячейки не редактируются
**Решение:**
1. Проверьте, что `initAllEditableTables()` вызвана
2. Проверьте консоль на ошибки
3. Попробуйте обновить страницу (Ctrl+F5)

### Проблема: Данные не сохраняются
**Решение:**
1. Проверьте, что функция `handleInput` определена
2. Убедитесь, что `saveData()` вызывается
3. Проверьте localStorage в DevTools
