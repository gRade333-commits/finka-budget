# 📝 Редактируемые таблицы - Инструкция

## 🎯 Что изменилось

Теперь **ВСЕ ячейки** во всех таблицах можно редактировать, включая:
- ✅ Номера строк
- ✅ Наименования
- ✅ Числовые данные
- ✅ Формулы
- ✅ Любые текстовые поля

## ➕ Новые возможности

### 1. Редактирование всех ячеек
Каждая ячейка теперь содержит текстовое поле (`<input>`), которое можно редактировать:
- Кликните в любую ячейку
- Введите или измените значение
- Данные автоматически сохраняются

### 2. Добавление новых строк
В каждой таблице появилась кнопка **"➕ Добавить новую строку"**:
- Нажмите кнопку под таблицей
- Появится новая пустая строка
- Заполните ячейки нужными данными
- Данные автоматически сохраняются

### 3. Удаление строк
У каждой строки есть кнопка **"🗑️"** (Удалить):
- Нажмите на кнопку удаления
- Подтвердите действие
- Строка и её данные будут удалены

## 🔧 Для разработчиков

### Подключение модуля

Для автоматической активации редактируемых таблиц на любой странице:

```html
<!-- Подключить скрипт -->
<script src="editable-table.js"></script>

<!-- В конце страницы или после загрузки таблиц -->
<script>
    // Автоматическая инициализация всех таблиц
    document.addEventListener('DOMContentLoaded', () => {
        initAllEditableTables('table', handleInput);
    });
</script>
```

### Параметры функции `initAllEditableTables`

```javascript
initAllEditableTables(selector, onInputCallback)
```

- **selector** (string): CSS селектор для поиска таблиц. По умолчанию `'table'`
- **onInputCallback** (function): Функция, вызываемая при изменении данных в ячейке

### Примеры использования

#### Пример 1: Инициализация всех таблиц
```javascript
initAllEditableTables();
```

#### Пример 2: С callback-функцией
```javascript
function handleInput(cellId) {
    console.log('Изменена ячейка:', cellId);
    saveData(); // Ваша функция сохранения
}

initAllEditableTables('table', handleInput);
```

#### Пример 3: Только определенные таблицы
```javascript
initAllEditableTables('.editable-table', handleInput);
```

### Функции модуля

#### `makeTableEditable(table, tableId, onInputCallback)`
Делает конкретную таблицу редактируемой.

```javascript
const table = document.getElementById('myTable');
makeTableEditable(table, 'myTable', handleInput);
```

#### `addTableRow(tableId, onInputCallback)`
Добавляет новую строку в таблицу.

```javascript
addTableRow('myTable', handleInput);
```

#### `deleteTableRow(rowId, tableId)`
Удаляет строку из таблицы.

```javascript
deleteTableRow('myTable_row_5', 'myTable');
```

#### `addAddRowButton(table, tableId, onInputCallback)`
Добавляет кнопку "Добавить строку" после таблицы.

```javascript
const table = document.getElementById('myTable');
addAddRowButton(table, 'myTable', handleInput);
```

## 🎨 Стилизация

Модуль автоматически добавляет стили для:
- `.editable-cell` - редактируемые ячейки
- `.btn-add-row` - кнопка добавления строки
- `.btn-delete-row` - кнопка удаления строки
- `.table-footer` - футер таблицы

Вы можете переопределить эти стили в своем CSS.

## 📋 Интеграция в существующие файлы

### Шаг 1: Подключить скрипт
```html
<script src="editable-table.js"></script>
```

### Шаг 2: Инициализировать после загрузки данных
```javascript
document.addEventListener('DOMContentLoaded', () => {
    loadData(); // Ваша функция загрузки
    
    // После загрузки данных
    setTimeout(() => {
        initAllEditableTables('table', handleInput);
    }, 100);
});
```

### Шаг 3: (Опционально) Убрать старую логику
Если в файле уже есть логика генерации редактируемых полей, можно:
- Оставить как есть (модуль не сломает существующую логику)
- Или убрать и полностью переключиться на модуль

## 🔍 Диагностика

### Проблема: Кнопки не появляются
```javascript
// Проверьте в консоли
console.log(document.querySelectorAll('.btn-add-row').length);
```

### Проблема: Данные не сохраняются
Убедитесь, что функция `handleInput` вызывает `saveData()`:
```javascript
function handleInput(cellId) {
    // ... ваш код ...
    saveData(); // Важно!
}
```

### Проблема: Таблицы не стали редактируемыми
Проверьте порядок выполнения:
```javascript
// ПРАВИЛЬНО - после загрузки данных
loadData().then(() => {
    initAllEditableTables();
});

// НЕПРАВИЛЬНО - до загрузки данных
initAllEditableTables();
loadData();
```

## 💡 Советы

1. **Резервное копирование**: Перед удалением строки данные автоматически удаляются из `localStorage`
2. **Автосохранение**: Используйте debounce в `handleInput` для оптимизации
3. **Валидация**: Добавьте проверку данных в `handleInput` перед сохранением
4. **Масштабируемость**: Для больших таблиц рассмотрите виртуализацию
