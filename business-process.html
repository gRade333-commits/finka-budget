🏛️ Система онлайн-бюджетирования РФМШ

📋 Описание проекта

Цель: Создать веб-систему для распределённого заполнения бюджетов филиалами с автоматической консолидацией данных для правления.

Текущая ситуация:

  • Два типа бюджетов: Республиканский бюджет (РБ) и Собственные средства (ПУ)
  • В каждом бюджете множество листов и таблиц
  • Сейчас заполняется только одним филиалом (Алмата)
  • Нужна консолидация данных от трёх филиалов

Филиалы участники:

Алмата Уральск Астана → Правление РФМШ

🔄 Бизнес-процесс

1

Заполнение филиалами

Экономисты каждого филиала заполняют онлайн-таблицы для своего филиала

  • Алмата → свои данные
  • Уральск → свои данные
  • Астана → свои данные
2

Сохранение в БД

Данные автоматически сохраняются в базе данных после каждого изменения

  • Автосохранение
  • История изменений
  • Валидация данных
3

Консолидация

Автоматическое суммирование данных от всех филиалов

  • Алмата + Уральск + Астана
  • По каждой статье
  • По каждому разделу
4

Отчёт для правления

Итоговые таблицы для правления РФМШ

  • Те же формы
  • Сводные данные
  • Экспорт в Excel

📊 Структура данных

Таблицы в базе данных:

Таблица Описание Ключевые поля
branches Справочник филиалов id, name, code
budget_types Типы бюджетов (РБ, ПУ) id, name, code
budget_sheets Листы бюджета id, budget_type_id, sheet_name, order
budget_items Статьи бюджета id, sheet_id, code, name, parent_id
budget_data Фактические данные branch_id, item_id, year, month, value
users Пользователи системы id, username, branch_id, role

👥 Роли пользователей

Роль Права доступа Функционал
Экономист филиала Редактирование данных своего филиала Заполнение таблиц, просмотр истории
Руководитель филиала Просмотр + утверждение данных филиала Утверждение бюджета, комментарии
Правление РФМШ Просмотр всех данных + консолидация Сводные отчёты, экспорт, аналитика
Администратор Полный доступ к системе Управление пользователями, настройки

✨ Функциональные требования

1. Онлайн-таблицы (идентичные Excel):

  • ✅ Точное повторение структуры Excel-файлов (строчка в строчку)
  • ✅ Автосохранение при изменении ячеек
  • ✅ Валидация данных (числовые поля, обязательные поля)
  • ✅ Выделение изменённых ячеек
  • ✅ Формулы и автоматические расчёты

2. Управление доступом:

  • ✅ Авторизация пользователей
  • ✅ Доступ только к своему филиалу (для экономистов)
  • ✅ Разделение прав по ролям
  • ✅ Журнал действий пользователей

3. Консолидация данных:

  • ✅ Автоматическое суммирование по филиалам
  • ✅ Обновление в реальном времени
  • ✅ Визуализация распределения по филиалам
  • ✅ Сравнение плановых и фактических данных

4. Отчётность:

  • ✅ Экспорт в Excel (форматы РБ и ПУ)
  • ✅ PDF-отчёты для печати
  • ✅ Графики и диаграммы
  • ✅ История изменений

⚠️ Важные моменты реализации:

  • Необходимо детально изучить структуру обоих Excel-файлов
  • Определить все формулы и связи между ячейками
  • Продумать логику консолидации для каждого типа статей
  • Настроить backup и восстановление данных

✅ План реализации:

  1. Этап 1: Анализ Excel-файлов и проектирование БД
  2. Этап 2: Создание backend API (Flask + PostgreSQL)
  3. Этап 3: Разработка frontend (HTML-таблицы)
  4. Этап 4: Реализация системы авторизации
  5. Этап 5: Механизм консолидации данных
  6. Этап 6: Экспорт и отчётность
  7. Этап 7: Тестирование и запуск

🚀 Следующие шаги

Для начала работы мне нужно:

  2. Определить какие листы и таблицы нужно реализовать в первую очередь
  3. Создать схему базы данных
  4. Разработать прототип одной таблицы для демонстрации
